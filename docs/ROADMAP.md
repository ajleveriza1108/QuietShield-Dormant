# QuietShield Dormant roadmap

## Completed in v0.2.0 Beta 1 R3

- Wireless pairing and helper restoration
- Accurate, user-friendly runtime states
- Expanded core and dynamic safety protection
- Per-app media and useful-background-work protection
- Close-sooner suggestions and optional automatic application for User Apps
- Low-frequency adaptive monitoring
- Local action history and three-day battery comparison
- Best-effort restart recovery
- Optimized beta build and size reporting
- Manufacturer compatibility guidance and beta report export

## Beta 2 priorities

- Physical-device test results for Samsung, Pixel, Xiaomi, Oppo, Vivo, Motorola, and Android tablets
- Wireless discovery reliability improvements based on real logs
- More precise playback and foreground-service parsing
- Safe restore behavior across restart, app update, screen lock, and network changes
- Battery and RAM profiling
- Accessibility, alarm, VPN, keyboard, launcher, and phone-service regression tests
- Release signing, licensing, updater, privacy policy, and store declarations

## Release candidate gates

- No core app can be slept, closed, or disabled
- Wireless pairing and restoration pass on supported Android 11+ devices
- Music and video survive all configured timers
- Helper loss is detected promptly
- Saved rules survive reboot and app update
- No measurable excessive background drain from Dormant
- Optimized APK reviewed if above 18 MB
- Every supported device family has a documented result or an explicit limitation
